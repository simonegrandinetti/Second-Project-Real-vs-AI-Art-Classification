\begin{abstract}
This project studies the binary classification of human-created and
AI-generated artwork using AI-ArtBench. The original dataset supports a richer
source and style attribution task, but here it is reformulated as a controlled
``real art versus fake art'' problem. The work compares five ImageNet-pretrained
transfer-learning configurations: two MobileNetV2 baselines, two ConvNeXt-Tiny
models, and a lightweight squeeze-and-excitation channel-attention variant on
top of ConvNeXt.
The protocol keeps the official test partition isolated, balances every split
by source and artistic style, and selects the final model only by validation
F1. The validation-selected ConvNeXt-Tiny achieved test accuracy
\BestTestAccuracy, F1 \BestTestFOne, and ROC-AUC \BestTestAUC. The headline
score is then qualified through subgroup analysis, perturbation robustness, an
independent holdout audit, and Grad-CAM visualizations.
\end{abstract}

\section{Introduction and motivation}

Generative image models have made the boundary between human-created and
machine-generated visual content less obvious. In art, this question is not
only technical. Many generated images imitate the texture, composition, and
style of historical artworks, while real artworks in digital datasets are
already mediated by photography, scanning, resizing, and compression. A
classifier can therefore learn something useful about image generation, but it
can also learn accidental traces of the data collection process. The main
purpose of this project is to build a detector while keeping those risks
visible.

The starting point is AI-ArtBench, introduced together with ArtBrain and an
AttentionConvNeXt architecture for explainable classification and attribution
of AI-generated art \cite{silva2023aiartbench,silva2024artbrain}. Instead of
attempting to reproduce the full paper, this coursework narrows the problem to
a binary task: human-created artwork versus artwork generated by AI systems.
That reformulation makes it possible to run a complete transfer-learning study
on a balanced subset, compare several design choices, and explain the model's
behaviour in a way that is still faithful to the data.

The guiding philosophy is conservative. The official test partition is never
used for early stopping or model selection. Human, Latent Diffusion, and Stable
Diffusion images are sampled with exact style quotas, so the model is not
rewarded merely for recognizing one overrepresented artistic movement. Results
are not reported as a single score alone: the report also asks whether the
score is stable across sources, styles, perturbations, and a second disjoint
test batch. This is important because a visually impressive detector may still
be exploiting shortcuts such as resolution differences, compression artefacts,
or generator-specific textures.

\section{Dataset and binary-task reformulation}

AI-ArtBench contains 185,015 images. The human subset consists of 60,000
artworks taken from ArtBench-10 \cite{liao2022artbench}; the AI subset contains
images generated by Latent Diffusion and Stable Diffusion models, both related
to the latent-diffusion family of high-resolution image generators
\cite{rombach2022latent}. The published split contains 155,015 training images
and 30,000 test images. The sources are not identical in their image formation
pipeline: human and Latent Diffusion images are provided at \(256\times256\),
whereas Stable Diffusion images are \(768\times768\). This matters because a
model could partly solve the task by recognizing acquisition or resolution
artefacts rather than a meaningful distinction between human and AI images.

The label used in this project is
\[
  y =
  \begin{cases}
    0, & \text{human-created artwork},\\
    1, & \text{AI-generated artwork}.
  \end{cases}
\]
Here \(y\) is the ground-truth target attached to each image. Latent Diffusion
and Stable Diffusion are therefore both positive-class sources, but they are
kept separate in the metadata. This separation is useful for two reasons.
First, it lets the training and test subsets include equal amounts of both
generator families. Second, it allows source-specific error analysis after
evaluation. If a detector is accurate on Stable Diffusion but weak on Latent
Diffusion, the binary F1 score alone would hide an important limitation.

The dataset also spans ten artistic styles: Art Nouveau, Baroque,
Expressionism, Impressionism, Post-Impressionism, Realism, Renaissance,
Romanticism, Surrealism, and Ukiyo-e. Style balancing is treated as part of the
experimental design, not as a cosmetic detail. Art styles differ strongly in
texture, colour, and composition, and a careless random sample could make the
classification problem easier or harder by changing the style distribution of
one class.

\section{Dataset preparation and split protocol}

The raw Kaggle directory is scanned into a metadata table containing image
path, source label, binary label, artistic style, file extension, and official
train/test partition. The scanner uses explicit folder-name aliases and fails
if unknown labels or unknown official splits are found. This is a deliberate
choice: silently accepting unexpected folder names would make the experiment
less reproducible than an immediate error.

All images are converted to RGB, resized to \(224\times224\), converted to
tensors, and normalized with ImageNet \cite{deng2009imagenet} mean and standard
deviation. In PyTorch, conversion to tensor maps pixel values from the integer
range \([0,255]\) to floating-point values in \([0,1]\). Let \(x_{cij}\) be the
tensor value at colour channel \(c\) and pixel location \((i,j)\), and let
\(\mu_c\) and \(\sigma_c\) be the fixed ImageNet mean and standard deviation
for channel \(c\). The normalized value is
\[
  x'_{cij} = \frac{x_{cij}-\mu_c}{\sigma_c},
\]
where \(x'_{cij}\) is the value actually given to the network. The ImageNet
statistics used here are
\[
  \mu=(0.485,\,0.456,\,0.406), \qquad
  \sigma=(0.229,\,0.224,\,0.225),
\]
for red, green, and blue respectively. They are not trainable model
parameters. They are fixed preprocessing constants estimated from the ImageNet
training distribution. Subtracting the mean recentres each channel around the
input distribution expected by the pretrained networks, and division by the
standard deviation puts the channels on a comparable scale. This matters
because the first layers of MobileNetV2 and ConvNeXt-Tiny were optimized under
exactly this convention; feeding unnormalized artwork images would shift the
activation statistics seen by those pretrained filters.

The ImageNet normalization is therefore a compatibility step for transfer
learning, not a claim that AI-ArtBench has the same distribution as ImageNet.
Resizing to 224 also matches the expected input size of the standard
pretrained architectures and removes the direct resolution difference from the
final tensor shape. It does not, however, prove that resolution-related traces
are absent; that is why a separate common-resampling robustness condition is
evaluated later.

The final protocol uses a deterministic seed-42 sample of 10,000 images:
\TrainCount{} for training, \ValidationCount{} for validation, and \TestCount{}
for the primary test set. Training and validation images are sampled only from
the official training partition. Test images are sampled only from the official
test partition. Every style receives the same per-source quota. In the primary
test set, for example, each style contributes 100 Human images, 50 Latent
Diffusion images, and 50 Stable Diffusion images, for a total of 200 images per
style.

This design creates three useful guarantees. The first is isolation: no image
path appears in more than one split. The second is source balance: the positive
class is not dominated by one generator. The third is style balance: the model
cannot obtain a strong score simply because one class has more examples of an
easy style. The resulting dataset is smaller than the full AI-ArtBench corpus,
but it is controlled enough to support clear model comparisons.

\section{Model background and implemented architectures}

All models in this project use transfer learning. The idea is that the early
and middle layers of a convolutional network trained on a large dataset such as
ImageNet already encode useful visual primitives: edges, textures, colour
contrasts, motifs, and increasingly complex shapes. Training a new classifier
from scratch on only 6,400 images would be unnecessarily unstable. Instead,
the pretrained backbone is reused and only part of the network is adapted to
the real-versus-AI task.

The pretrained weights come from the standard \texttt{torchvision} model
registry. MobileNetV2 \cite{sandler2018mobilenetv2} uses
\texttt{MobileNet\_V2\_Weights.DEFAULT}, and ConvNeXt-Tiny
\cite{liu2022convnext} uses \texttt{ConvNeXt\_Tiny\_Weights.DEFAULT}; both are
ImageNet-trained checkpoints distributed with torchvision's model definitions.
The implementation also contains a ResNet18 builder with
\texttt{ResNet18\_Weights.DEFAULT} for compatibility and smoke tests, but
ResNet18 is not part of the reported E0--E4 experiment matrix. Checkpoints
loaded during replication are not downloaded pretrained models: they are the
best validation-F1 checkpoints saved by this project's training runs.

The E4 variant uses squeeze-and-excitation (SE) channel gating
\cite{hu2018squeeze}. This is the same basic channel-reweighting idea as in
the SE paper, but attached only to the final ConvNeXt feature map in this
project.

The following subsections use compact arrow schematics. They are not separate
architectures from the implemented PyTorch models; they are readable summaries
of the actual data flow. A block arrow means that the tensor produced by one
component is passed to the next component, while the surrounding prose states
which parts are frozen and which parts are trainable.

\subsection{One-logit binary head}

Every architecture ends in a single scalar logit \(z=f_\theta(x)\), where
\(x\) is the preprocessed image tensor, \(f_\theta\) is the network, and
\(\theta\) denotes its trainable parameters. The probability assigned to the
fake/AI class is
\[
  p_\theta(y=1\mid x) = \sigma(z) = \frac{1}{1+\exp(-z)}.
\]
Here \(\sigma\) is the sigmoid function and \(p_\theta(y=1\mid x)\) is the
model's estimated probability that \(x\) is generated. The fixed decision rule
is \(\hat{y}=1\) if \(p_\theta\geq0.5\), otherwise \(\hat{y}=0\). Training
minimizes binary cross-entropy,
\[
  \mathcal{L}_{\mathrm{BCE}}(y,z)
  = -y\log\sigma(z) - (1-y)\log(1-\sigma(z)).
\]
In this loss, \(y\) is the ground-truth binary label and \(z\) is the logit
before the sigmoid. Using a single logit is simpler than a two-output softmax
head and is exactly suited to this binary reformulation. It also makes the
Grad-CAM interpretation straightforward: positive logits support the fake
class, while negative logits support the real class.

\subsection{MobileNetV2 baseline}

MobileNetV2 is used as the lightweight baseline.
Its design is built around efficiency. A standard convolution learns filters
that mix spatial neighbourhoods and channels at the same time. MobileNetV2
factorizes this operation. First, a depthwise convolution learns one spatial
filter for each input channel. Then a \(1\times1\) pointwise convolution mixes
information between channels. This depthwise-separable design is much cheaper
than a standard convolution and is the reason MobileNetV2 is attractive as a
fast baseline.

The main repeated block is the inverted residual with a linear bottleneck.
The block starts from a compact feature representation and expands it to a
higher-dimensional internal space. The depthwise convolution is applied in
that expanded space, where the block has more channels available for
intermediate computations. A final \(1\times1\) projection compresses the
features back to a lower-dimensional bottleneck. The residual connection,
when input and output shapes match, lets the block learn a correction to the
input instead of having to relearn the entire representation.

The word ``inverted'' is important. In older residual blocks, a wide
representation is often compressed and then expanded again. MobileNetV2 does
the opposite: it starts from a compact representation, expands it internally,
applies the depthwise convolution in the expanded space, and projects back to a
low-dimensional bottleneck. The final projection is linear rather than passed
through another nonlinearity, because excessive nonlinear compression in a
small bottleneck can destroy information. Residual connections are used when
the input and output shapes match, helping gradients flow through the network.

At the level of one MobileNetV2 block, let \(x\) denote the input feature
tensor and let \(\Delta x\) denote the transformation learned inside the
block. The operation can be summarized as
\[
  x
  \rightarrow
  1{\times}1\ \text{channel expansion}
  \rightarrow
  3{\times}3\ \text{depthwise spatial convolution}
  \rightarrow
  1{\times}1\ \text{linear projection}
  \rightarrow
  x+\Delta x .
\]
The last residual addition is present only when the spatial size and number of
channels match. This detail is one reason MobileNetV2 can be both deep and
efficient: most computation happens in cheap depthwise operations, while the
pointwise layers control how information is mixed across channels.

For this project, the ImageNet classifier is replaced by a single-logit binary
head. The resulting architecture can be read in two parts: a pretrained
feature extractor that maps the \(224\times224\) RGB image to a compact visual
representation, and a newly initialized linear classifier that maps that
representation to the real/fake logit. In the frozen setting, all
convolutional features are fixed and only the classifier receives gradient
updates. This makes MobileNetV2 a clean baseline: it asks how much information
about real versus generated art is already present in generic ImageNet
features. E0 uses this baseline without augmentation, and E1 uses the same
architecture with augmentation. The comparison isolates whether mild image
transformations help the classifier generalize.

\[
  \begin{aligned}
  &224{\times}224\ \text{RGB image}
  \rightarrow
  \text{ImageNet normalization}
  \rightarrow
  \text{MobileNetV2 feature extractor} \\
  &\rightarrow
  \text{global pooling}
  \rightarrow
  \text{binary linear head}.
  \end{aligned}
\]

\subsection{ConvNeXt-Tiny}

ConvNeXt-Tiny is the stronger backbone in the experiment matrix. ConvNeXt
modernizes convolutional networks using design ideas that made vision
transformers competitive, while keeping a convolutional inductive bias. In
practical terms, it is still a convolutional network: it
builds a hierarchy of feature maps whose spatial resolution decreases while
the number of channels increases. Earlier stages tend to represent local
edges, colours, and textures; later stages represent more abstract visual
patterns. This hierarchy is attractive for artwork because the task may depend
both on low-level texture artefacts and on higher-level rendering patterns.

Compared with older convolutional designs, ConvNeXt changes several details
that matter in practice. It begins with a patch-like stem that reduces spatial
resolution early, somewhat like the patch embedding used by vision
transformers. Its blocks use large depthwise convolution kernels, which give
each block a wider spatial view while keeping the computation manageable. The
network also uses Layer Normalization (LayerNorm) \cite{ba2016layernorm} and
Gaussian error linear unit (GELU) activations \cite{hendrycks2016gelu}, which
are common in modern deep architectures. Channel mixing is handled by
pointwise layers arranged in an inverted-bottleneck style: channels are
expanded, transformed, and projected back. The result is still a convolutional
feature hierarchy, but with design choices closer to contemporary high-capacity
vision models.

For an introductory reading, the unfamiliar terms in the previous paragraph
can be unpacked as follows. The patch-like stem is simply the first
downsampling convolution: it turns the \(224\times224\) image into a smaller
grid of local feature vectors, so later blocks do not process every raw pixel
independently. A large-kernel depthwise convolution applies one spatial filter
per channel, giving each channel a wider neighbourhood while avoiding the cost
of a full convolution that mixes every input channel with every output channel.
The channel expansion/projection pair is similar in spirit to a small
per-location multilayer perceptron: first the feature vector is made wider,
then it is compressed back to the block's output width.

LayerNorm normalizes a feature vector inside each individual example rather
than using statistics from the whole mini-batch. If \(h_c\) is channel \(c\)
of a feature vector \(h\), then
\[
  \operatorname{LayerNorm}(h)_c
  =
  \gamma_c
  \frac{h_c-\mu_h}{\sqrt{\sigma_h^2+\epsilon}}
  + \beta_c .
\]
Here \(\mu_h\) and \(\sigma_h^2\) are the mean and variance of the channels in
that feature vector, \(\epsilon\) prevents division by zero, and
\(\gamma_c,\beta_c\) are learned scale and shift parameters. This differs from
batch normalization because it does not depend on other images in the same
mini-batch, which is convenient when batch sizes are modest and when
fine-tuning pretrained models.

GELU is the nonlinearity used inside the ConvNeXt block. For a scalar input
\(u\), it is
\[
  \operatorname{GELU}(u)=u\,\Phi(u)
  \approx
  \frac{1}{2}u
  \left[
    1+\tanh\!\left(\sqrt{\frac{2}{\pi}}\,(u+0.044715u^3)\right)
  \right],
\]
where \(\Phi(u)\) is the cumulative distribution function of a standard
normal random variable. Intuitively, GELU behaves like a smooth version of a
rectified activation: large positive inputs are mostly kept, large negative
inputs are mostly suppressed, and values near zero are changed gradually
rather than by a hard cutoff.

A typical ConvNeXt block uses the same residual notation: \(x\) is the input
feature tensor and \(\Delta x\) is the block's learned update. It can be read
as
\[
  \begin{aligned}
  x
  &\rightarrow
  \text{large-kernel depthwise convolution}
  \rightarrow
  \text{LayerNorm} \\
  &\rightarrow
  \text{channel expansion}
  \rightarrow
  \operatorname{GELU}
  \rightarrow
  \text{channel projection}
  \rightarrow
  x+\Delta x .
  \end{aligned}
\]
The large depthwise kernel gives the block spatial context, while the
expansion and projection layers perform channel mixing. This is why ConvNeXt
is still a convolutional model, but one whose internal blocks resemble some of
the successful design habits of transformer-era vision networks.

The ``Tiny'' version is the smallest member of the ConvNeXt family, but it is
still substantially more expressive than the MobileNetV2 baseline. That makes
it a good middle ground for this coursework: large enough to test whether a
modern backbone helps, but small enough to fine-tune on the available GPU
without turning the project into a full-scale architecture search.

In the implementation, the pretrained ConvNeXt-Tiny classifier is replaced by
a single-output linear layer. The rest of the model keeps the torchvision
ConvNeXt-Tiny feature hierarchy. Two training modes are tested. In E2, the
backbone is frozen and only the classifier is updated. This measures the
quality of ConvNeXt as a generic visual feature extractor. In E3, the final
ConvNeXt stage and classifier are trainable. In torchvision's ConvNeXt-Tiny,
this final stage is the last high-level feature block, closest to the
classifier and farthest from raw pixels. This is a compromise between
head-only training and full fine-tuning: the early and middle layers remain
stable, while the high-level features closest to the decision boundary can
adapt to AI-art-specific patterns.

The project-level ConvNeXt flow is therefore
\[
  \begin{aligned}
  &224{\times}224\ \text{RGB image}
  \rightarrow
  \text{ImageNet normalization}
  \rightarrow
  \text{patch-like ConvNeXt stem} \\
  &\rightarrow
  \text{ConvNeXt stages}
  \rightarrow
  \text{final normalization and classifier}
  \rightarrow
  \text{binary logit}.
  \end{aligned}
\]

\subsection{The ConvNeXt-SE channel-attention variant}

The E4 model is a controlled squeeze-and-excitation channel-attention variant,
not a removal study. It starts from the same torchvision ImageNet-pretrained
ConvNeXt-Tiny feature extractor used by E3 and adds a small SE module after the
final ConvNeXt feature map. It is also not a reproduction of the full
AttentionConvNeXt architecture. The purpose is narrower: isolate whether
image-dependent final-channel reweighting adds useful information on top of
the E3 last-stage fine-tuning setup. The implemented model takes the final
ConvNeXt feature map
\(F\in\mathbb{R}^{C\times H\times W}\), where \(C\) is the number of channels
and \(H\times W\) is the spatial resolution. For each channel \(k\), it
computes a global average-pooled channel statistic \(s_k\):
\[
  s_k = \frac{1}{HW}\sum_i\sum_j F_{kij},
\]
where \(F_{kij}\) is the activation of channel \(k\) at spatial position
\((i,j)\). The vector \(s\) is then passed through a small gating network:
\[
  a = \sigma\!\left(W_2\,\delta(W_1s)\right).
\]
Here \(W_1\) and \(W_2\) are the two learned linear layers of the SE multilayer
perceptron, \(\delta\) is the ReLU nonlinearity, and \(\sigma\) is the sigmoid
that maps gates to values between 0 and 1. The gated feature map is
\(F' = a\odot F\), where \(a\) is broadcast across spatial positions and
\(\odot\) denotes elementwise multiplication. This is the core idea of SE:
let the network reweight feature channels according to the current image. In
this project the SE block is attached only to the last feature block. It tests
whether final-channel attention helps the binary task, while staying
computationally feasible and honest about the architecture actually
implemented.

Its implemented flow is
\[
  \begin{aligned}
  &\text{ConvNeXt final feature map } F
  \rightarrow
  \text{squeeze / global average pooling}
  \rightarrow
  \text{SE gate MLP} \\
  &\rightarrow
  \text{channel reweighting } F'
  \rightarrow
  \text{binary classifier}.
  \end{aligned}
\]

Interpreting the SE block is easiest if channels are viewed as learned visual
detectors. Some channels may respond to brush-like textures, others to smooth
synthetic gradients, borders, colour transitions, or high-level motifs. Global
average pooling summarizes how strongly each detector fired over the whole
image. The two linear layers then learn channel gates between 0 and 1. A gate
near 1 preserves a channel; a gate near 0 suppresses it. Because the gates are
computed separately for every image, the model can emphasize different
channels for different styles or sources. In E4, only the last ConvNeXt stage,
the SE block, and the classifier are trainable. This keeps the experiment
close to E3 while testing whether this extra channel-selection mechanism adds
value.

The measured result should be read with this limited role in mind. E3 already
fine-tunes the last ConvNeXt stage, so the high-level features closest to the
classifier can already adapt to real-versus-generated cues. The SE block then
acts only as a lightweight channel gate on those final features: it uses global
average pooling, has no spatial attention, and cannot localize borders,
brush-like textures, or generator artefacts by itself. Because it only rescales
channels, the following classifier can often compensate for small changes in
the gates, so the network is not strongly perturbed. At the same time, the
selected ConvNeXt configuration is already close to the performance ceiling on
this controlled split, leaving little room for a small final-channel gate to
increase F1. The observed SE delta, \AttentionDelta, is therefore best
interpreted as a practical non-improvement for this setup, not as evidence
that attention mechanisms are generally useless.

\section{Training philosophy and optimization}

The five experiments are designed as controlled comparisons rather than as an
open-ended hyperparameter search. E0 and E1 compare MobileNetV2 without and
with augmentation. E1 and E2 compare a lightweight backbone with ConvNeXt-Tiny
under the same augmentation setting. E2 and E3 isolate the effect of
fine-tuning the last ConvNeXt stage. E3 and E4 test whether the simplified SE
block improves the selected ConvNeXt model.

Training uses AdamW \cite{loshchilov2019adamw}, which decouples weight decay
from the adaptive gradient update. In simplified form, the update can be read
as an Adam step plus a separate shrinkage term. If \(\theta_t\) denotes the
trainable parameters at step \(t\), the update is
\[
  \theta_{t+1}
  =
  \theta_t
  - \eta
  \left(
    \frac{\hat{m}_t}{\sqrt{\hat{v}_t}+\epsilon}
  \right)
  - \eta\lambda\theta_t .
\]
Here \(\eta\) is the learning rate, \(\hat{m}_t\) and \(\hat{v}_t\) are Adam's
bias-corrected first and second moment estimates, \(\epsilon\) is a small
stability constant, and \(\lambda\) is the weight-decay coefficient. This is
useful in transfer learning because most trainable parameters belong to newly
initialized heads or final blocks, and regularization should not be entangled
with the adaptive gradient statistics. The learning rate is \(10^{-3}\) for
head-only training and \(10^{-4}\) for partial fine-tuning. The smaller
fine-tuning rate is used because pretrained features should be adjusted
carefully rather than overwritten.

Augmentation is deliberately mild: random resized crops, horizontal flips,
small colour jitter, and occasional autocontrast. The goal is to reduce
dependence on exact framing and colour statistics, not to create unrealistic
paintings. Strong augmentations could damage stylistic information that is
actually relevant to the task. Early stopping monitors validation F1 and stops
after three epochs without improvement. The best checkpoint is the one with
the highest validation F1, and the test set remains untouched until that
choice has already been made.

\section{Evaluation metrics and model selection}

The main scalar metric is F1. Let \(\mathrm{TP}\), \(\mathrm{FP}\), and
\(\mathrm{FN}\) denote true positives, false positives, and false negatives
with respect to the generated-art class. Precision measures how many predicted
generated images are really generated, and recall measures how many generated
images were found. F1 combines them as
\[
  F_1 =
  \frac{2\,\mathrm{precision}\,\mathrm{recall}}
       {\mathrm{precision}+\mathrm{recall}},
\]
where
\[
  \mathrm{precision}=\frac{\mathrm{TP}}{\mathrm{TP}+\mathrm{FP}},
  \qquad
  \mathrm{recall}=\frac{\mathrm{TP}}{\mathrm{TP}+\mathrm{FN}}.
\]
F1 is appropriate because false positives and false negatives are both
important: a real artwork incorrectly flagged as AI is as meaningful as an AI
image incorrectly accepted as human. Accuracy is also reported because the
test subset is balanced. The receiver operating characteristic area under the
curve (ROC-AUC) is included as a threshold-independent summary of how well
fake images tend to receive higher scores than real images \cite{fawcett2006roc}.
The confusion matrix remains useful because it shows the direction of
mistakes, not only their total number.

The source-level analysis uses a different reporting rule. A source group
contains only one binary class: Human is always real, while Latent Diffusion
and Stable Diffusion are always fake. Computing source-level F1 for such a
single-class group would be misleading. The report therefore uses the
source-appropriate error rate: false-positive rate for Human and
false-negative rate for each AI generator. Style groups, in contrast, contain
both classes and can be evaluated with the full binary metric set.

Robustness testing applies controlled image perturbations to the selected
model after training is complete. The checkpoint, decision threshold, and test
image paths are kept fixed; only the input image transformation changes. In
implementation terms, each perturbation is applied first, and the result is
then passed through the same evaluation pipeline used for clean testing:
resize to \(224\times224\), tensor conversion, and ImageNet normalization.
This ordering matters because all robustness conditions still reach the model
with the same final tensor shape.

The first family contains photometric perturbations: contrast, brightness, and
colour saturation. Contrast factors of \(0.5\), \(1.5\), and \(2.0\) either
flatten or exaggerate tonal differences. Brightness factors of \(0.7\) and
\(1.3\) darken or brighten the whole image. Saturation factors of \(0.5\) and
\(1.5\) weaken or strengthen colour intensity. These tests probe whether the
detector depends too strongly on exact colour and luminance statistics. In
artwork, that is relevant because digitization, display settings, and image
editing can easily alter these properties without changing the semantic
content of the picture.

The second family round-trips the image through JPEG compression at qualities
90, 50, and 20. Quality 90 is a light compression condition, while 50 and 20
introduce progressively stronger block artefacts, smoothing, and high-frequency
detail loss. This test asks whether the detector remains stable under a very
common distribution shift: images being saved, uploaded, or recompressed before
classification. A detector that relies heavily on delicate pixel-level
artefacts may degrade sharply as JPEG quality decreases.

The third family applies blur and noise. Gaussian blur with radius 1 or 2
removes local high-frequency detail, which is useful for checking whether the
classifier depends on fine texture traces. Deterministic Gaussian noise with
standard deviation 0.03 or 0.08 adds small random fluctuations before
normalization. The noise is seeded from the image content, so the same image
always receives the same perturbation even if dataloader workers or batch order
change.

The fourth family is common resampling. Every image, regardless of its original
source, is first resized with bicubic interpolation to \(128\times128\) or
\(64\times64\), and only then resized by the standard evaluation transform to
\(224\times224\). This creates the same low-resolution bottleneck for Human,
Latent Diffusion, and Stable Diffusion images. It is not part of the
replication audit and it is not mixed with the independent holdout split. Its
purpose is specifically to test whether the classifier is exploiting
source-resolution or resizing shortcuts, since the dataset contains different
native resolutions across sources. If performance collapsed under common
resampling, it would suggest that part of the detector's behaviour depends on
resolution-related artefacts rather than more robust visual cues.

\section{Grad-CAM explainability}

Gradient-weighted class activation mapping (Grad-CAM) \cite{selvaraju2017gradcam}
is included as an additional qualitative analysis, not as proof that the model
reasons like a human. The method asks a precise question: for a chosen class
score, which spatial regions in the final convolutional feature map had the
strongest positive influence? It is especially suitable here because all
tested backbones preserve spatial feature maps before the final classifier.

Let \(A^k\) be channel \(k\) of the selected target layer's activation map, let
\(A_{ij}^k\) be its value at spatial position \((i,j)\), and let \(y^c\) be
the score for the class \(c\) being explained. Grad-CAM computes a channel
importance weight by spatially averaging the gradient of the score with
respect to that activation map:
\[
  \alpha_k^c =
  \frac{1}{Z}\sum_i\sum_j
  \frac{\partial y^c}{\partial A_{ij}^k}.
\]
Here \(Z\) is the number of spatial positions in the activation map, so the
double sum is an average over the feature-map grid. The heatmap is then
\[
  L^c_{\mathrm{GradCAM}} =
  \mathrm{ReLU}\left(\sum_k \alpha_k^c A^k\right).
\]
Here \(L^c_{\mathrm{GradCAM}}\) is the class-\(c\) heatmap and \(A^k\) is the
whole spatial map for channel \(k\). The ReLU keeps only positive evidence for
the chosen score. The resulting low-resolution map is upsampled to
\(224\times224\) and overlaid on the input image.

The actual workflow in this project follows those equations directly. First,
the final spatial block is selected: the last ConvNeXt stage for ConvNeXt
models, the last MobileNetV2 feature block for MobileNetV2, and the last
ResNet block for the compatibility model. A forward hook stores activations
during the forward pass. A backward hook stores gradients during
backpropagation. The model receives one preprocessed image. For fake-class
explanations, the backpropagated score is the binary logit. For real-class
explanations, the score is the negative logit, because lower logits support
the real class. The gradients are averaged over spatial positions, used as
channel weights, and combined with the activations to form the heatmap.

The panels in Figures~\ref{fig:gradcam-correct} and
\ref{fig:gradcam-misclassified} should be read with care. A bright region
means that the selected layer was locally sensitive there for the chosen
prediction. It does not prove that the model recognized a brushstroke,
understood composition, or found a causal artefact. The value of Grad-CAM is
more modest but still useful: it can reveal whether the model repeatedly looks
at borders, signatures, texture fields, or object-like regions, and it can
help identify suspicious shortcuts worth testing quantitatively.

\section{Limitations}

The main limitation is distributional. AI-ArtBench contains two related
diffusion sources, not every possible image generator. A high score on this
benchmark does not imply reliable detection of Midjourney, DALL-E, edited
photographs, screenshots, or future diffusion systems. The human images are
digitized artworks, while the AI images are born digital, and the native
resolutions differ by source. The protocol reduces these shortcuts through
balanced sampling and common preprocessing, and the robustness tests probe
some of them, but it cannot remove every dataset-specific trace.

The second limitation is scale. The controlled 10,000-image subset makes the
model-comparison study practical and interpretable, but it uses only part of
the full 185,015-image dataset. The independent replication audit adds another 2,000
official-test images and improves confidence in same-distribution stability,
but it is still sampled from the same benchmark. Finally, explainability
remains qualitative. Grad-CAM can suggest where the model is sensitive, but it
does not by itself establish what visual concept the model has learned.

\section{Conclusion}

This project builds a controlled binary detector for human versus AI-generated
art under the official AI-ArtBench split. The best measured configuration is a
ConvNeXt-Tiny model with mild augmentation and last-stage fine-tuning, reaching
held-out F1 \BestTestFOne{} and ROC-AUC \BestTestAUC. The controlled
comparisons show that the stronger backbone and partial fine-tuning matter more
than the simplified SE channel-attention insertion. The subgroup,
robustness, replication, and Grad-CAM analyses then put the headline result in
context: the detector is strong under the chosen benchmark protocol, but it is
not a universal art-authentication system.

\appendix
\section{Compact protocol pseudocode}

\subsection*{Split construction}
\begin{enumerate}[leftmargin=*]
  \item Scan all images and infer source, style, binary label, and official
  split from the path.
  \item Validate that the pinned AI-ArtBench inventory matches the expected
  source counts.
  \item For each requested split, sample exact source/style quotas with a
  fixed seed.
  \item Draw training and validation only from the official training pool;
  draw test only from the official test pool.
  \item Reject the split if any image path appears in more than one partition.
\end{enumerate}

\subsection*{Training and model selection}
\begin{enumerate}[leftmargin=*]
  \item Build the selected ImageNet-pretrained backbone and replace its head
  with one binary logit.
  \item Freeze the backbone, unfreeze the final stage, or train the SE
  attention variant according to the experiment definition.
  \item Train with AdamW and binary cross-entropy.
  \item After each epoch, compute validation F1.
  \item Save the checkpoint with the best validation F1 and stop early after
  three stale validation epochs.
  \item Evaluate the restored best checkpoint on the held-out test set.
\end{enumerate}

\subsection*{Replication audit}
\begin{enumerate}[leftmargin=*]
  \item Exclude every original train, validation, and primary-test path.
  \item Sample a second official-test holdout with the same source/style quotas.
  \item Reload each saved checkpoint without retraining.
  \item Evaluate clean training predictions and clean replication predictions
  using the standard resize-normalize transform.
  \item Bootstrap F1 within source/style strata and report percentile
  intervals for replication F1 and F1 gaps.
\end{enumerate}

\subsection*{Grad-CAM computation}
\begin{enumerate}[leftmargin=*]
  \item Register hooks on the final spatial feature block.
  \item Run one image forward and store activations.
  \item Backpropagate the predicted-class score: logit for fake, negative
  logit for real.
  \item Average gradients spatially to obtain channel weights.
  \item Combine weighted activations, apply ReLU, upsample, and overlay the
  heatmap on the image.
\end{enumerate}
